# 前言

平台工程标志着 DevOps 开发的下一个地震性转变，通过将持续改进与长期自动化联系起来。它将多年的宝贵知识浓缩成一个统一的框架，以演进组织构建、监督和扩展其数字平台的方式。在当今快节奏的商业环境中，快速将产品推向市场可能使公司成功或失败。创建有机定制的平台通常会浪费时间并错失机会。平台工程提供了一个简单而强大的解决方案：专注于创新和价值，同时将基础设施和转型的复杂性委托给专家。有效的平台工程之路并非一条直线；它充满了需要深刻理解技术和战略的挑战。

许多组织怀着最好的意图开始这段旅程，但发现自己陷入了低效率的循环，需要 12-18 个月或更长时间才能逃脱，如果能够逃脱的话。事实是，你需要的东西已经存在；挑战在于知道如何利用、整合和委派平台责任。

这本书不仅仅是一本理论汇编；它是一本实用的指南，提供了可操作的见解、最佳实践和经过实战检验的工具。你将学习如何精确配置可扩展、安全和可靠的平台，确保你的组织保持领先。到结束时，你将感到完全准备好领导你所在组织的平台工程实施。

这本全面的指南涵盖了从奠定文化和架构基础到融入 AI 驱动平台最新进展的各个方面。通过实用的教程、动手项目和深入的自我评估问题，你将掌握平台工程，并能够在你所在的组织中带头采用它。主要特点包括以下内容：

+   设计和构建一个有效的平台。

+   建立平台基础设施指标和集成，包括 AI 解决方案。

+   通过加速交付和持续收入最大化平台成功。

到你读完这本书的时候，你不仅会为未来做好准备——你还将准备好去塑造未来。这本书将成为你在创建一个 DevOps 兼容、AI 驱动的平台道路上的盟友，这个平台不仅是一个技术奇迹，也是一个战略资产，将你的组织置于数字创新的前沿。

本书专为 IT、产品和企业职能领域的专业人士编写，他们是企业转型的前沿。它直接面向商业领袖、IT 部门的转型变革推动者、软件开发者和负责确保平台稳定性、可靠性和整体健康的 IT 运维团队。无论你是领导转型、开发平台还是确保运营效率，本书旨在为你提供成功所需的知识和工具。

本书为那些厌倦了不断循环构建和维护自己平台的软件开发者提供了一条更好的路径。在平台构建（通常持续 40-60 周）上花费的时间是从创建推动投资回报率和增强竞争优势的应用程序中转移出来的时间。如果你的目标是构建在市场上取得成功的应用程序，那么明智的做法是利用由专家设计的平台，这样你可以专注于真正重要的事情：创新和交付。同样，对于 IT 运维团队来说，平台工程可以减轻平台维护的负担，让你能够专注于增强平台整体健康的战略任务。

本书是 C 级高管、经理和工程、安全、平台和运营团队总监，以及准备通过采用平台工程的战略方法提升其组织的软件开发者和平台运营商不可或缺的指南。

# 本书涵盖内容

*第一章*，*现代平台工程简介*，提供了关于平台为何至关重要的基础解释，以及平台面临的一些基本问题，如上市速度和资源成本，以及一些现有的软件解决方案。本章通过现有工具的比较分析继续进行。了解 DevOps 是什么，可以让人通过使用平台来集中关注流程、反馈和交付等重要方面。

*第二章*，*架构基础和策略*，帮助你从基本的云迁移开始理解平台标准，包括从 AWS 到 Azure 或 Google 的云格式，以及云结构，以及设置 Kubernetes 集群。基本标准引导人们设计和实现这些空间内的定制架构，但架构也带来了自己的标准。

*第三章*, *文化转型与领导力*，表明平台不仅仅是关于对生产系统进行技术变革，而是关于支持实施的变革的文化变革。文化变革需要承认现状并建立通往组织变革的道路。作为前沿创新，本章塑造了技术领导层的决策，通过有效的变革达到平台市场，提供用户价值。

*第四章*, *平台工程生态系统*，探讨了平台工程不仅仅是安装单个软件修复或实现配置代码化，而是调整整个交付生态系统。优秀的平台会整合多种不同的工具以实现成功。将这些工具整合到平台的不同方面是有效平台的关键要素。此外，这些工具应通过 DevOps 方法融入 CI/CD 成果中。

*第五章*, *将人工智能融入平台工程*，讨论了如何通过使用生成式 AI 模型在编码和自动化软件开发操作中帮助从不同位置摄取大量数据。本章突出了可以集成以支持基于 AI 的成果，从而增强整体能力的多种现成工具。用户必须认识到自己的 AI 方法，以及竞争对手可能如何使用 AI 及其经验教训。

*第六章*, *平台数据管理工程*，展示了平台不仅需要初始形成，还需要在多个应用程序中管理数据。必须实施有效的策略，优化这些策略，然后管理数据。介绍了数据管理工具以及处理所有数据的方法，从用户输入到流式解决方案。

*第七章*, *安全、合规性和风险管理*，探讨了 DevOps 如何被重新定义为 DevSecOps，但任何运营解决方案都需要安全。平台强调在部署前提供默认安全解决方案，用于漏洞跟踪、软件物料清单和补丁解决方案。安全不仅限于平台本身，还包括管理连接到平台的人员，尤其是在具有 AI 工具的 PaaS 和 SaaS 类型解决方案中。

*第八章*, *现实世界应用与案例研究*，让您回顾展示成功平台转型和平台与业务战略整合的案例研究。它探讨了失败的平台转型，以突出战略规划和在企业中的颠覆的价值。使用 DevOps 成功模型来捕捉以往的经验教训，并将它们纳入当前运营的支持。本章还考察了那些尚未完全实施的技术和解决方案可能带来的潜在未来经验教训。

*第九章*, *测试、质量保证和运营*，强调平台解决方案在基础之上构建，提供标准化的管道和测试流程。成功实施这些流程会产生可重复的证据体，使所有软件都能按照相同的标准进行衡量。实施这些标准在整个产品中创造质量，并允许持续的成功。

*第十章*, *构建高性能平台团队*，表明平台的核心是拥有有效的平台团队。采用最优的购买与构建策略，组织可以在保持可扩展性和可靠性的同时，最小化专门的平台人员。在 DevOps 框架内，可以设计高性能团队，而不需要基于平台从标准化工具、生成式 AI 解决方案和现代化的可观察性产生的平台效益的专门知识。

*第十一章*, *从愿景到现实：掌握企业平台工程*，强调愿景型领导力、战略规划和持续改进在推动平台成功中的重要性。它探讨了如何向利益相关者阐述平台倡议的价值，衡量和跟踪关键绩效指标，以及根据不断变化的企业和技术环境调整策略。它包括实际案例和实用建议，以保持平台作为适应性、弹性和持续发展的资产，从而推动业务价值。
