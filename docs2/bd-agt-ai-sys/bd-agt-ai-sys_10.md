

# 第十章：常见用例和应用

在我们之前对基于 LLM 的代理系统中的风险和挑战的考察基础上，从对抗性攻击到伦理问题，我们现在将注意力转向它们的实际应用。本章探讨了代理系统如何通过结合 LLM 和目标导向的行为以及自主决策能力来改变各个领域。我们将看到这些代理如何理解上下文，制定计划，并采取行动以实现特定目标，同时与人类保持有意义的互动。

在我们探索这些应用时，我们将关注代理如何利用 LLM，不仅作为语言处理器，还作为核心推理引擎，它使不同领域的复杂规划和执行成为可能。这代表了从传统人工智能系统到根本性的转变，因为这些代理现在可以调整其行为，从互动中学习，并在与人类意图保持一致的同时，以越来越高的自主性运行。本章首先探讨了基于 LLM 的代理在创意和艺术领域的变革性影响，接着是它们在自然语言处理和对话系统中的进步。然后深入探讨这些代理在机器人和自主系统中的集成，最终聚焦于它们在决策支持和优化中的作用。

+   创意和艺术应用

+   决策支持和优化

# 创意和艺术应用

基于 LLM 的代理在创意领域的集成标志着超越简单生成性人工智能工具的显著演变。本节探讨了代理系统如何通过理解上下文、保持创意方向和与人类艺术家积极合作的能力来改变创意工作流程。与仅生成内容的传统人工智能工具不同，这些代理可以进行持续的创意对话，适应不断变化的要求，并同时平衡多个艺术和技术约束。 

## 创意和艺术代理的演变

创意和艺术代理的发展代表了 AI 系统对创意过程贡献方式的范式转变。早期的生成 AI 工具主要侧重于根据预定义的提示产生静态输出，提供有限的交互和适应性。然而，现代的代理系统已经发展到积极参与动态和迭代的创意过程。它们利用高级能力，如上下文保留、自适应学习和多模态集成，作为真正的合作伙伴而不是被动工具。这些代理不仅生成内容，还理解艺术意图，根据反馈优化输出，并与人类合作伙伴的愿景保持一致。这种演变突显了 AI 发展的一个更广泛趋势，即从静态生成转向交互式、上下文感知和自适应系统，以前所未有的方式补充和增强人类创造力。代理系统正在产生影响的重点领域包括以下方面：

+   **艺术协作**：能够通过保持一致的风格、艺术意图和主题连贯性跨多个迭代来协助艺术家的代理，使人类创造力和 AI 生成的增强无缝集成

+   **音乐创作**：旨在理解和应用音乐理论、适应不同流派，并与表演者或作曲家实时协作以创作和谐创新的作曲的系统

+   **写作和叙事发展**：能够与作者共同创作，保持角色一致性、情节连贯性和叙事流畅性的代理，在保持作者独特声音和叙事风格的同时提供创意建议

## 真实世界的应用

在创意领域中，代理系统实际应用的实施已经开始显示出有希望的结果。Adobe 的 Firefly 代理系统展示了多个专业代理如何能够在复杂项目中保持创意的一致性。在这个系统中，不同的代理协同工作：一个维护品牌身份和风格指南，另一个确保在各种媒体中保持资产的一致性表示，第三个处理技术规范和格式要求。这代表了从简单的生成工具到维护创意上下文并实时适应用户反馈的协作系统的重大演变。

同样，环球音乐集团的 AI 音乐制作系统展示了代理系统如何在音乐创作中保持创造性的连贯性。该系统采用专门针对音乐创作不同方面的代理：旋律代理理解音乐主题和动机，和声代理保持音调一致性，编排代理处理乐器编排。这些代理在创作过程中协同工作，同时保持作曲家的创作意图和风格偏好，展示了多个代理如何在保持艺术视野的同时协作完成复杂创意任务。

为了更好地理解代理系统如何改变复杂的创意工作流程，让我们从电影行业考察一个详细的使用案例。此示例展示了多个代理如何协作弥合创意视野与技术执行之间的差距，这是一个长期困扰电影制作预可视化过程的挑战。

### 问题陈述

电影导演和分镜画家在预可视化序列上花费大量时间进行迭代，需要多个部门之间持续的沟通以将创意视野与技术可行性对齐。传统工具缺乏理解和适应创意意图的能力，同时考虑技术限制。目标是创建一个多代理系统，帮助将导演的创意视野转化为技术上可行的预可视化序列，同时保持艺术一致性和制作限制。

代理系统方法如下：

+   **导演代理**：

    +   处理场景的自然语言描述

    +   维护整体创意视野和风格一致性

    +   将艺术意图传达给其他参与者

+   **技术** **监督代理**：

    +   评估技术可行性

    +   考虑预算和资源限制

    +   当需要时提出替代方案

+   **可视化代理**：

    +   生成初始分镜脚本和 3D 预览

    +   根据其他代理的反馈调整输出

    +   在场景间保持视觉一致性

代理通过以下方式协同工作：

+   通过 LLM 能力实现共享的上下文理解

+   创意需求和技术要求之间的持续反馈循环

+   对创意方向或技术限制变化进行实时适应

### 环境和外部系统

多代理系统在以下生产环境中运行：

+   **资产管理数据库**：存储 3D 模型、纹理和之前的分镜脚本

+   **制作管理系统**：跟踪预算、进度和资源分配

+   **渲染农场 API**：管理 3D 可视化的计算资源

+   **摄像机和设备数据库**：技术规格和可用性

+   **参考库**：过往作品、风格指南和情绪板档案

+   **版本控制系统**：维护迭代和变更的历史记录

+   **协作平台**：允许团队成员实时反馈

### 为什么这更好

当前的预可视化工具在孤立状态下运行，需要人类中介在创意和技术需求之间进行翻译。代理系统方法提供以下优势：

+   创意愿景与技术可行性之间的持续对齐

+   通过实时协作减少迭代周期

+   在适应约束的同时保持创意一致性

+   通过自动化技术验证提高资源利用效率

这种方法突出了代理系统如何超越简单的内容生成，积极参与到创意决策过程中，以细微的理解处理多个约束和目标。他们在预可视化方面的成功凸显了它们在需要复杂协作、迭代优化和适应不断变化的艺术愿景的创意领域的广泛适用性。通过保留上下文、理解意图和平衡竞争优先级，这些系统正成为不可或缺的创意伙伴，而不仅仅是工具。

在探索代理系统对创意工作流程的变革性影响后，我们现在将重点转向代理系统最突破性的应用之一：自然语言处理和对话界面。这些系统擅长理解细微的语言，在复杂的对话中保持上下文，并有效地执行任务，对于促进有意义的 人机交互至关重要——这是我们将在下一节深入探讨的主题。

# 自然语言处理和对话代理

在探索创意应用的基础上，我们现在转向基于 LLM（大型语言模型）的代理可能最自然的领域：语言理解和对话。虽然传统的语言模型可以处理和生成文本，但代理系统增加了关键能力：在长时间交互中保持上下文，通过对话执行复杂任务，并根据用户需求和反馈调整其响应。

## 语言代理的演变

今天的对话代理代表了超越简单聊天机器人或虚拟助手的重大飞跃。这些系统可以维持复杂的对话，理解细微的上下文，并通过自然对话执行复杂任务。例如，Anthropic 的 Claude 和 OpenAI 的 ChatGPT 可以在保持多个回合的一致性的同时进行详细的技术讨论，根据用户的技能水平和先前互动调整其响应。区分现代语言代理和传统聊天机器人的关键能力如下：

+   **上下文保留**：现代语言代理擅长保持连贯和有意义的讨论，即使在对话跨越多个主题和会话时也是如此。这种记住先前交流的能力使他们能够随着时间的推移提供更加个性化和上下文相关的响应。

+   **任务执行**：这些智能体不仅超越了简单的问答功能，还能将自然语言指令转化为可执行步骤，从而实现诸如日程安排、数据检索或系统配置等领域的无缝任务完成。这种能力弥合了沟通与执行之间的差距。

+   **自适应交互**：现代智能体能够根据用户偏好、专业水平或情境需求动态调整其沟通风格、语气和复杂性，从而为不同受众提供更具吸引力和可访问性的交互。

+   **多模态理解**：与传统聊天机器人不同，现代智能体可以处理和响应文本、图像、代码和结构化数据的组合，使它们能够处理更复杂的查询，并整合多种形式的信息，以更丰富、更细腻地理解用户请求。

## 现实世界应用

当前实施展示了语言智能体在不同领域的多功能性。例如，Salesforce 的 Agentforce（以前称为 Einstein 虚拟助手）通过在多个客户交互中保持上下文并实时访问相关数据库信息，帮助客户服务代表。同样，GitHub 的 Copilot Chat 可以在执行相关开发任务的同时维持关于代码的技术讨论。让我们看看一个涉及企业知识管理系统的用例。

### 问题陈述

大型组织在跨部门的知识碎片化方面面临挑战，这使得员工难以有效访问和利用机构知识。传统的搜索系统和文档无法捕捉不同信息片段之间的上下文和联系。

为了解决问题，创建一个多智能体系统，该系统能够理解、组织和检索企业知识，同时保持上下文并处理跨越多个领域和文档的复杂查询。

这里是智能体系统方法：

+   **查询理解智能体**：

    +   处理自然语言问题

    +   识别隐含的上下文和要求

    +   将复杂查询分解为子任务

+   **知识导航智能体**：

    +   将不同知识源之间的关系进行映射

    +   在多份文档中保持上下文

    +   跟踪信息来源

+   **响应合成智能体**：

    +   结合来自多个来源的信息

    +   根据用户角色调整响应细节级别

    +   在多个交互中保持一致性

智能体通过以下方式协作：

+   对组织上下文的共同理解

+   从用户交互中持续学习

+   动态适应不同用户角色和需求

### 环境和外部系统

知识管理系统与多个企业系统和数据源接口：

+   **内部系统**：

    +   **文档管理系统**：SharePoint、Confluence 和内部维基

    +   **通信平台**：Slack 和 MS Teams

    +   **项目管理工具**：Jira 和 Asana 的项目文档

    +   **电子邮件服务器**：存档的电子邮件线程和附件

    +   **代码仓库**：GitHub 和 GitLab 的文档和讨论

    +   **人力资源系统**：培训材料和政策文件

    +   **客户关系管理**：客户互动历史和支持工单

+   **外部 API 和服务**：

    +   **行业新闻 API**：Bloomberg 或 Reuters 的市场更新

    +   **研究数据库**：学术论文和专利数据库

    +   **监管数据库**：合规性文件和法律更新

    +   **云存储**：用于共享文档的 Google Drive 或 OneDrive

    +   **翻译服务**：用于多语言文档处理

    +   **网络监控**：社交媒体和竞争对手网站

+   **访问控制层**：

    +   基于角色的访问管理系统

    +   安全分类数据库

    +   用户身份验证服务

    +   审计日志系统

    +   合规性监控工具

### 为什么这更好

传统的知识管理系统依赖于精确的关键词匹配和预定义的分类。代理方法提供了一些优势：

+   复杂查询的自然语言理解

+   基于上下文的信息检索

+   相关信息的动态连接

+   根据用户角色和专业知识提供个性化响应

这种知识管理方法展示了代理系统如何改变组织内部的信息访问和利用，使机构知识更易于获取和执行。正如我们所见，语言代理如何革命性地改变信息访问和通信，我们现在将注意力转向它们在机器人和自主系统中的物理世界应用。将语言理解与物理控制相结合为人类-机器人交互创造了新的可能性，我们将在下一节中探讨。

# 机器人和自主系统

从基于语言的交互转向物理世界应用，我们现在探讨基于 LLM 的代理如何改变机器人和自主系统。虽然传统机器人依赖于预编程的行为和刚性控制系统，但代理系统使机器人能够理解自然语言指令，对其环境进行推理，并动态调整其行为。这种语言模型与物理控制系统的集成代表了机器人与人类及其环境互动方式的根本转变。

## 机器人代理的演变

机器学习与机器人的结合创造了能够弥合人类意图与物理行动之间差距的系统。与遵循固定规则的传统机器人系统不同，现代机器人代理能够理解上下文，从经验中学习，并在保持与人类目标一致的同时做出自主决策。以下是区分现代机器人代理的关键能力：

+   物理任务的自然语言理解

+   关于环境约束的上下文推理

+   对变化条件的实时适应

+   从演示和反馈中学习

+   视觉、语言和控制的多元集成

## 现实世界应用

当前实施展示了代理系统在各个领域的多功能性。在 Boston Dynamics，Atlas 等机器人展示了基于语言指令如何被转换为复杂的物理动作。同样，加州大学伯克利分校的机器人系统展示了生成模型如何使机器人能够实时适应杂乱的环境，而麻省理工学院的 RoboBrain 系统则说明了代理如何通过利用庞大的知识库创造性地解决物理操作任务。让我们考虑一个实际案例，并评估一个代理系统如何帮助实施这样的系统。

### 问题陈述

传统制造单元难以应对产品变化和意外中断。当前系统需要大量重新编程以适应新产品，并且无法有效处理意外情况，如设备故障或供应链中断。操作人员必须不断干预以管理变化并优化工作流程。目标是创建一个多代理系统，协调灵活的制造单元，能够适应产品变化，处理中断，优化流程，同时保持质量标准和安全协议。

让我们看看代理系统的方法：

+   **规划和协调代理**：

    +   理解自然语言生产需求

    +   开发和调整制造序列

    +   协调不同机器人系统之间的工作

    +   维护整体生产目标

+   **机器人控制代理**：

    +   将高级指令转换为运动原语

    +   管理实时传感器反馈

    +   根据环境变化调整动作

    +   确保安全的人机交互

+   **质量和优化代理**：

    +   实时监控生产质量

    +   提出流程改进建议

    +   预测维护需求

    +   优化资源利用

+   **异常处理代理**：

    +   检测异常和中断

    +   生成恢复策略

    +   管理意外的人为干预

    +   在异常情况下保持安全协议

代理通过以下方式协作：

+   对制造背景的共享理解

+   实时传感器数据集成

+   流程优化的连续反馈循环

+   在中断期间动态重新分配任务

### 环境和外部系统

制造系统与多个系统接口：

+   **制造基础设施**：

    +   **机器人臂和末端执行器**：具有不同能力的多种机器人类型

    +   **视觉系统**：用于零件识别和质量控制的摄像头和 3D 传感器

    +   **PLC 系统**：用于设备控制的**可编程逻辑控制器**（**PLC**）

    +   **物料搬运系统**：输送带、**自动导引车**（**AGV**）和存储系统

+   **信息系统**：

    +   **制造执行系统**（**MES**）：生产调度和跟踪

    +   **企业资源计划**（**ERP**）：资源和库存管理

    +   **质量管理系统**（**QMS**）：质量数据和规范

    +   **数字孪生平台**：实时模拟和预测

    +   **维护管理系统**：设备健康监控

+   **外部接口**：

    +   **供应链管理系统**：物料可用性和物流

    +   **客户订单系统**：产品规格和要求

    +   **合规数据库**：安全标准和监管要求

    +   **知识库**：历史生产数据和最佳实践

### 为什么这更好

传统的机器人制造系统依赖于严格的编程，并且需要大量的人为干预以进行适应。采用代理系统方法提供了几个变革性的优势：

+   生产变更和问题解决的自然语言交互

+   无需重新编程即可实时适应产品变化

+   自主处理中断和异常

+   基于历史和实时数据的持续优化

+   当需要时无缝集成人力专业知识

+   积极的质量控制和维护预测预测

这种方法通过使机器人能够理解上下文、适应变化并在保持与生产目标和安全要求一致的同时做出自主决策，从根本上改变了制造灵活性。代理系统在制造中的成功展示了它们在复杂物理环境中的潜力，在这些环境中，多个系统必须协调并适应不断变化的情况。通过结合语言理解和物理控制以及实时优化，这些系统正从简单的自动化工具发展成为制造过程中的智能合作伙伴。

在我们结束对机器人和自主系统的探索后，我们将注意力转向代理系统如何通过决策支持和优化改变业务运营。在保持业务上下文的同时处理复杂数据流的能力使这些系统在战略决策方面特别有价值，我们将在下一节中探讨这一点。

# 决策支持和优化

在探索了代理系统在物理世界中的应用后，我们现在转向它们在增强人类决策和解决复杂优化问题中的作用。虽然传统的决策支持系统依赖于固定规则和静态分析，但基于 LLM 的代理能够理解上下文，权衡利弊，并在保持与业务目标和约束一致的同时提供适应性建议。

## 决策支持代理的演变

将 LLMs 与决策支持系统集成已经改变了组织处理信息和做出战略选择的方式。现代决策代理可以分析多个数据流，理解复杂商业环境，并在保持对组织目标和约束的认识的同时生成可操作的见解。区分现代决策支持代理的关键能力如下：

+   多模式数据分析与综合

+   基于上下文的推荐生成

+   实时适应变化条件

+   解释推理和权衡

+   将领域专业知识与数据驱动见解相结合

## 现实世界应用

当前实施展示了代理系统在各个领域的多功能性：

+   **金融行业**：

    +   摩根大通（JPMorgan Chase）的 LOXM 系统分析市场数据、新闻和社交媒体以识别投资机会

    +   两 sigma 的 Venn 系统将市场分析与强化学习结合用于投资策略

+   **医疗保健**：

    +   密歇根大学（University of Michigan）的 HealthPal 通过分析病历和遗传数据提供个性化的治疗方案推荐

    +   斯坦福大学（Stanford）的 DeepPill 系统根据患者档案和病史优化药物治疗方案

+   **工业应用**：

    +   西门子（Siemens）的 MindSphere 通过实时分析和适应优化制造流程

    +   埃克森美孚（ExxonMobil）的能源展望使用预测建模进行长期资源规划

现代供应链的复杂性提供了一个引人入胜的例子，说明了代理系统如何转型传统业务流程。让我们考察一个现实世界的实施案例，其中多个代理共同协作以协调全球供应链网络，展示这些系统如何在保持业务目标的同时处理复杂的多利益相关者环境。

### 问题陈述

全球供应链面临前所未有的复杂性，涉及多个利益相关者、可变提前期和频繁的中断。传统的优化工具难以处理现代供应链的动态性，并且无法有效平衡如成本、速度和可持续性等竞争目标。人类规划者被数据量以及所需决策的速度所淹没。在这里，我们旨在创建一个多代理系统，该系统能够协调端到端供应链优化，能够在平衡多个目标的同时进行实时决策并适应中断。

代理系统方法如下：

+   **战略规划代理**：

    +   分析市场趋势和需求模式

    +   制定长期采购策略

    +   平衡成本、风险和可持续性目标

    +   保持与业务目标的协调一致

+   **操作优化代理**：

    +   管理日常物流运营

    +   优化路线和资源分配

    +   处理实时调度调整

    +   协调多个承运人和仓库

+   **风险管理代理**：

    +   监测全球事件和中断

    +   评估对供应链运营的影响

    +   生成应急计划

    +   提供早期预警信号

+   **可持续性优化代理** :

    +   跟踪环境影响指标

    +   优化以减少碳足迹

    +   建议替代路线和采购

    +   确保符合环境法规

代理通过以下方式协作：

+   对供应链背景的共享理解

+   实时数据集成和分析

+   持续的情景规划和风险评估

+   根据变化条件进行动态再优化

### 为什么这样做更好

传统的供应链优化系统在有限的背景下运行，难以适应快速变化。一种代理系统方法提供了几个变革性的优势：

+   用于战略发展和问题解决的天然语言交互

+   实时适应多个变化的变量

+   积极的风险识别和缓解

+   在多个竞争目标之间进行平衡优化

+   在战略决策点整合人力专业知识

+   从历史决策中持续学习和改进

这种方法通过实现智能、情境感知的决策，考虑多个利益相关者和目标，同时保持与业务目标和可持续性要求的一致性，从根本上转变了供应链管理。

代理系统在供应链优化中的成功展示了它们在复杂商业环境中的潜力，在这些环境中，必须平衡多个因素，同时适应不断的变化。通过结合语言理解和高级优化技术以及实时适应，这些系统正从简单的决策支持工具发展到值得信赖的战略顾问。

在本章中，我们探讨了四个关键领域，在这些领域中代理系统正在重塑可能性：创意应用、自然语言处理、机器人和决策支持。虽然这些领域展示了显著的发展，但它们仅代表了代理系统产生影响的部分领域。从教育、科学研究到医疗保健和环境保护，这些系统在众多未在本章中涵盖的领域找到了新颖的应用。

我们探索的现实世界应用——从协作电影预可视化到制造单元编排，以及从企业知识管理到供应链优化——展示了代理系统正成为有价值的合作伙伴，而不仅仅是工具。它们表明，人工智能的未来不在于取代人类能力，而在于通过智能协作增强它们。

# 问题

1.  创意应用中的代理系统与传统生成 AI 工具有何不同？请以本章中的例子进行讨论。

1.  在制造单元编排的背景下，解释多个代理如何协作以及为什么这种方法比传统自动化系统更有效。

1.  比较和对比在机器人学和决策支持系统中代理系统的作用。它们在维护上下文方面的方法有何不同？

1.  使用本章中的企业知识管理用例，解释不同代理如何协作解决复杂的信息检索问题。

1.  本章讨论了供应链优化网络作为用例。在此背景下使用代理系统方法的关键优势是什么，以及不同的代理如何相互补充？

# 答案

1.  与仅生成内容的传统工具不同，创意应用中的代理系统在多个迭代中维护上下文和创意愿景。例如，在电影预可视化用例中，导演代理在与技术总监和可视化代理协作时保持创意愿景，以确保艺术完整性和技术可行性。这使创意目标和实际约束之间的持续一致成为可能，这是传统生成工具无法实现的。

1.  在制造单元编排中，四个专业代理（规划和协调、机器人控制、质量和优化以及异常处理）通过共享上下文理解和连续反馈循环协同工作。这种方法更有效，因为它能够实时适应产品变化和中断，同时保持质量标准。传统的自动化系统缺乏这种灵活性，需要人为干预以应对变化或意外情况。

1.  在机器人技术中，代理系统专注于将语言理解转化为物理动作，同时保持安全和操作限制。例如，机器人控制代理将高级指令转化为运动原语，同时确保安全的人机交互。**运动原语**是机器人技术中的基本运动模式，如抓取或旋转，用作复杂动作的构建块。一个**机器人控制代理**将这些高级命令转化为这些原语，以确保在保持操作限制的同时安全且高效地执行。

    在决策支持系统中，代理专注于处理多个信息流，同时保持业务上下文和战略目标。虽然两者都保持上下文，但机器人代理必须弥合物理-数字鸿沟，而决策支持代理必须平衡多个竞争目标。

1.  在企业知识管理系统，三个代理协同工作：

    +   查询理解代理处理自然语言问题并识别隐含的上下文

    +   知识导航代理映射不同信息源之间的关系

    +   响应合成代理在适应用户角色的同时结合信息

    这些代理共享组织上下文，并从用户交互中学习，使它们能够比传统搜索系统更有效地处理跨越多个领域和文档的复杂查询。

1.  供应链优化网络通过四个专业代理（战略规划、运营优化、风险管理以及可持续性优化）展示了代理系统的优势，这些代理协同工作以平衡多个目标。关键优势包括用于战略发展的自然语言交互、实时适应变化、主动识别风险以及跨竞争目标进行平衡优化。这些代理通过维护供应链环境的不同方面——从长期战略到日常运营、风险管理以及可持续性目标——相互补充。
